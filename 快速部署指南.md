# 🚀 快速部署到 GitHub Pages

## 三步完成部署

### 步骤 1️⃣：在 GitHub 创建仓库

1. 访问 https://github.com/new
2. 仓库名称：`telecom-signup`（或你喜欢的名字）
3. 设为 **Public**（公开）
4. **不要**勾选 "Add a README file"
5. 点击 **Create repository**

### 步骤 2️⃣：推送代码

打开 PowerShell，在项目目录运行：

```powershell
# 替换为你的 GitHub 用户名
$USERNAME = "your-github-username"
$REPO = "telecom-signup"

# 添加远程仓库
git remote add origin https://github.com/$USERNAME/$REPO.git

# 重命名分支并推送
git branch -M main
git push -u origin main
```

**或者使用部署脚本：**

```powershell
.\deploy.ps1 -Username "your-github-username" -RepoName "telecom-signup"
```

### 步骤 3️⃣：启用 GitHub Pages

1. 进入你的 GitHub 仓库页面
2. 点击 **Settings** → **Pages**（左侧菜单）
3. **Source** 选择：`Deploy from a branch`
4. **Branch** 选择：`main` → `/(root)`
5. 点击 **Save**

### ✅ 完成！

等待 1-2 分钟，你的网站就会上线：

```
https://your-github-username.github.io/telecom-signup/
```

## ❓ 常见问题

**Q: 推送时提示认证失败？**  
A: 使用 GitHub Personal Access Token 代替密码，或配置 SSH 密钥。

**Q: Pages 显示 404？**  
A: 等待 1-2 分钟，GitHub 需要时间构建。检查 Branch 设置是否正确。

**Q: 如何更新已部署的网站？**  
A: 修改代码后执行 `git add . && git commit -m "更新" && git push`

**Q: 可以部署到其他平台吗？**  
A: 可以！本项目是纯静态 HTML，可部署到 Vercel、Netlify、Gitee Pages 等任何静态托管服务。
